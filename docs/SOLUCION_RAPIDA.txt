🔍 CAUSA ENCONTRADA:
═══════════════════════════════════════════════════════════════════════════════

La función getFileType() en script.js era muy restrictiva y fallaba cuando:

1. El navegador no reconocía el MIME type del archivo
2. Enviaba MIME type genérico: "application/octet-stream"
3. El código buscaba palabras clave en MIME (pero no las encontraba)
4. No recurría a verificar la extensión del archivo como fallback

RESULTADO: Rechazaba el archivo ❌

═══════════════════════════════════════════════════════════════════════════════

✅ SOLUCIÓN IMPLEMENTADA:
═══════════════════════════════════════════════════════════════════════════════

Reescribí completamente la función getFileType() con 5 niveles de validación:

NIVEL 1: Verifica la EXTENSIÓN del archivo (primero - más confiable)
├─ .pdf, .doc, .docx, .xls, .xlsx, .ppt, .pptx, .txt, .csv, .odt, .ods
└─ ✅ Si coincide → "document"

NIVEL 2: Verifica MIME type completo conocido
├─ application/pdf
├─ application/msword
├─ application/vnd.openxmlformats-officedocument.wordprocessingml.document
├─ application/vnd.ms-excel
├─ application/vnd.ms-powerpoint
└─ ✅ Si coincide → "document"

NIVEL 3: Busca palabras clave en MIME type
├─ pdf, word, excel, sheet, powerpoint, presentation, text
└─ ✅ Si contiene → "document"

NIVEL 4: Verifica nuevamente por extensión (último recurso)
├─ Extrae la extensión del nombre del archivo
├─ Compara con lista de extensiones conocidas
└─ ✅ Si coincide → "document"

NIVEL 5: Si nada funciona → Rechaza (null)
