🔍 CAUSA RAÍZ:
═══════════════════════════════════════════════════════════════════════════════

La función getFileType() era muy restrictiva:

CÓDIGO ORIGINAL (ROTO):
┌─────────────────────────────────────────────────────────────────────────┐
│ getFileType(mimeType, fileName) {                                       │
│     if (mimeType.startsWith('image/')) return 'image';                  │
│     if (mimeType.startsWith('video/')) return 'video';                  │
│                                                                         │
│     // Aquí está el problema:                                          │
│     if (mimeType.includes('pdf') || fileName.endsWith('.pdf'))         │
│         return 'document';                                              │
│     if (mimeType.includes('word') || ...)                              │
│         return 'document';                                              │
│     ...                                                                 │
│     return null;  // ← RECHAZA CUALQUIER TIPO NO RECONOCIDO            │
│ }                                                                        │
└─────────────────────────────────────────────────────────────────────────┘

EL PROBLEMA ESPECÍFICO:
─────────────────────────────────────────────────────────────────────────
1. Los navegadores a veces NO envían el MIME type correcto
2. El código buscaba palabras clave en MIME type, pero:
   - Si el navegador no lo reconoce → mimeType es "application/octet-stream"
   - Luego checkea con .includes('pdf') → pero si mimeType es vacío falla
3. Solo como último recurso checkea la extensión del archivo

EJEMPLO QUE FALLABA:
─────────────────────────────────────────────────────────────────────────
Si intentabas subir: documento.pdf
- mimeType: "application/octet-stream" (genérico)
- fileName: "documento.pdf"

FLUJO ROTO:
┌─────────────────────────────────────────────────────────────────────┐
│ 1. ¿Es image/? NO                                                   │
│ 2. ¿Es video/? NO                                                   │
│ 3. ¿mimeType.includes('pdf')? NO (es application/octet-stream)    │
│ 4. ¿mimeType.includes('word')? NO                                   │
│ 5. ... más checks que fallan ...                                    │
│ 6. return null ← ¡RECHAZADO!                                        │
│                                                                     │
│ RESULTADO: "Tipo de archivo no soportado"                          │
└─────────────────────────────────────────────────────────────────────┘

═══════════════════════════════════════════════════════════════════════════════

✅ SOLUCIÓN IMPLEMENTADA:
═══════════════════════════════════════════════════════════════════════════════

Cambi la lógica para priorizar la EXTENSIÓN del archivo (más confiable):

NUEVO CÓDIGO (REPARADO):
┌─────────────────────────────────────────────────────────────────────────┐
│ getFileType(mimeType, fileName) {                                       │
│     console.log(`Analizando: ${fileName}...`);                          │
│                                                                         │
│     // 1. Imágenes y videos por MIME (confiables)                       │
│     if (mimeType && mimeType.startsWith('image/'))                      │
│         return 'image';                                                 │
│     if (mimeType && mimeType.startsWith('video/'))                      │
│         return 'video';                                                 │
│                                                                         │
│     // 2. DOCUMENTOS POR EXTENSIÓN (primero - más confiable)           │
│     const docExtensions = /\.(pdf|doc|docx|txt|xls|xlsx|ppt|pptx|...) │
│     if (docExtensions.test(fileName))                                   │
│         return 'document';  ← FUNCIONA AUNQUE MIME SEA GENÉRICO        │
│                                                                         │
│     // 3. Documentos por MIME type completo                            │
│     const documentMimeTypes = [                                         │
│         'application/pdf',                                              │
│         'application/msword',                                           │
│         'application/vnd.openxmlformats-...'                           │
│         ...                                                             │
│     ];                                                                  │
│     if (mimeType && documentMimeTypes.includes(mimeType))              │
│         return 'document';                                              │
│                                                                         │
│     // ... más checks ...                                              │
│                                                                         │
│     // 4. ÚLTIMO RECURSO: Por extensión conocida                      │
│     const ext = fileName.split('.').pop().toLowerCase();               │
│     if (['pdf', 'doc', 'docx', 'xls', 'xlsx', 'ppt', ...])           │
│         return 'document';                                              │
│                                                                         │
│     return null;  // Solo rechaza si NADA coincide                    │
│ }                                                                        │
└─────────────────────────────────────────────────────────────────────────┘

🎯 CAMBIOS ESPECÍFICOS REALIZADOS:
═══════════════════════════════════════════════════════════════════════════════

1. AGREGA ACEPTACIÓN EN EL INPUT:
   fileInput.setAttribute('accept', '*/*');
   └─ Permite seleccionar cualquier tipo de archivo

2. PRIORIZA EXTENSIÓN DEL ARCHIVO:
   const docExtensions = /\.(pdf|doc|docx|...)$/i;
   if (docExtensions.test(fileName)) return 'document';
   └─ Primero verifica la extensión (80% de confiabilidad)

3. LISTA COMPLETA DE MIME TYPES:
   documentMimeTypes = ['application/pdf', 'application/msword', ...]
   └─ Reconoce todos los MIME types oficiales

4. MEJOR MANEJO DE ERRORES:
   if (error.name === 'QuotaExceededError') {
       // Almacenamiento lleno
   }
   └─ Avisa si superas los 50 MB

5. LOGGING DETALLADO:
   console.log(`Analizando: ${fileName}, MIME: ${mimeType}`);
   └─ Te muestra qué está pasando en la consola
